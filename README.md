3D Data Processing: deal_3D:

The A3DK08 folder contains the original .fid data, and the Hyper_complex_data folder contains the corresponding complex .mat data with imaginary and real parts.

First, in nmrpipe, the proc_dirct function is used to convert the original .fid data into .ft1 data (the original .fid 3D data is in hypercomplex form; .ft1 is essentially performing a Fourier transform on the direct dimension (and likely phase modulation), then only the real part is retained). Then, the hyper_complex_data_proc code... The `hyper2complex` function transforms hypercomplex data into a real part and an imaginary part (both complex numbers). Then, `find_peak_3D_without_limit` slices the 3D data along its direct dimension, resulting in corresponding 2D slices that are undersampled for reconstruction. The reconstructed results are then clustered along their direct dimension to form 3D data. Finally, `complex2hyper` from `hyper_complex_data_proc`, combined with `proc_indirct` in `nmrpipe`, yields the reconstructed, zero-padded, windowed .mat data.
